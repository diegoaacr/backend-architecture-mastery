# What is architecture

## 🎯 Objetivo

Entender qué es la arquitectura de software y por qué es importante.

---

## 🧠 Explicación simple

La arquitectura es la forma en que organizas un sistema.

Define cómo se estructuran las partes y cómo se comunican entre sí.

---

## 🧩 Ejemplo

Imagina una aplicación:

* Tiene frontend
* Tiene backend
* Tiene base de datos

La arquitectura define:

👉 cómo se conectan  
👉 qué hace cada parte  
👉 cómo se organizan  

---

## 🏗️ Qué define la arquitectura

La arquitectura decide:

* Cómo dividir el sistema
* Cómo se comunican los componentes
* Dónde va cada responsabilidad
* Cómo escalar el sistema

---

## 🔁 Sin arquitectura vs con arquitectura

Sin arquitectura:

* Código desordenado
* Difícil de mantener
* Difícil de escalar

Con arquitectura:

* Código organizado
* Fácil de entender
* Más fácil de escalar

---

## 💡 Idea clave

La arquitectura no es código.

Es cómo organizas el código.

---

## ⚠️ Errores comunes

* Pensar que arquitectura es solo usar frameworks
* Copiar arquitecturas sin entenderlas
* Sobre-ingenierizar desde el inicio

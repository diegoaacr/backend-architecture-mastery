# Contributing

Gracias por querer contribuir a este repositorio.

---

## 🎯 Objetivo

Mantener el contenido claro, simple y útil para aprender arquitectura backend de forma progresiva.

---

## 🚀 Cómo contribuir

1. Haz fork del repositorio
2. Crea una rama (`feature/nombre-del-cambio`)
3. Realiza tus cambios
4. Abre un Pull Request

---

## 🧠 Reglas de contenido

* Explicaciones simples y directas
* 1 concepto por archivo
* Evitar teoría innecesaria
* Mantener consistencia con el resto del repo
* No duplicar contenido
* Usar ejemplos claros (cuando aplique)

---

## 📝 Formato de los archivos

Cada archivo debe seguir esta estructura:

* Título
* Objetivo
* Explicación simple
* Ejemplo (si aplica)
* Idea clave
* Errores comunes / buenas prácticas (si aplica)
* Siguiente paso

---

## 🧩 Enfoque del repositorio

Este repositorio está enfocado en:

* Entender conceptos
* Pensar en arquitectura
* No memorizar teoría

---

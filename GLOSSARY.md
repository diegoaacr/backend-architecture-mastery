# Glossary

Términos básicos de arquitectura backend explicados de forma simple.

---

## Backend

Parte del sistema que maneja la lógica, datos y procesamiento detrás de la aplicación.

---

## API

Interfaz que permite que sistemas se comuniquen entre sí.

---

## Server

Programa o máquina que recibe peticiones y responde con datos.

---

## Database

Lugar donde se almacenan los datos de una aplicación.

---

## Request

Petición que un cliente envía al backend.

---

## Response

Respuesta que el backend devuelve al cliente.

---

## Architecture

Forma en que se organiza un sistema.

---

## Layer

Nivel dentro de una aplicación con una responsabilidad específica.

---

## Coupling

Nivel de dependencia entre partes del sistema.

---

## Cohesion

Qué tan bien una parte del sistema cumple una sola responsabilidad.

---

## Monolith

Aplicación donde todo está en un solo proyecto.

---

## Microservices

Sistema dividido en múltiples servicios independientes.

---

## Module

Parte organizada de un sistema que agrupa funcionalidades relacionadas.

---

## Design Pattern

Solución común a un problema de diseño en software.

---

## Clean Architecture

Forma de organizar el código separando negocio, aplicación e infraestructura.

---

## Domain

Reglas de negocio principales del sistema.

---

## Use Case

Acción específica que el sistema puede ejecutar.

---

## Repository

Capa encargada de acceder a datos.

---

## Entity

Objeto que representa algo importante del negocio.

---

## Value Object

Objeto sin identidad, definido solo por sus valores.

---

## Event

Mensaje que representa que algo ocurrió en el sistema.

---

## Message Broker

Sistema que permite enviar mensajes entre servicios.

---

## API Gateway

Punto de entrada que gestiona todas las peticiones hacia los servicios.

---

## Load Balancer

Distribuye tráfico entre múltiples servidores.

---

## CI/CD

Proceso automatizado para integrar, probar y desplegar código.

---

## Docker

Herramienta para empaquetar aplicaciones en contenedores.

---

## VPS

Servidor virtual donde se ejecuta una aplicación.

---

## Cloud

Infraestructura remota donde se ejecutan sistemas.

---

## Serverless

Modelo donde no gestionas servidores directamente.

---

## Authentication

Proceso de verificar quién eres.

---

## Authorization

Proceso de definir qué puedes hacer.

---

## JWT

Token usado para autenticar usuarios.

---

## Cache

Almacenamiento temporal para mejorar rendimiento.

---

## Redis

Base de datos rápida usada como cache.

---

## Observability

Capacidad de entender qué está pasando en el sistema.

---

## Logging

Registro de eventos que ocurren en el sistema.

---

## Metrics

Datos numéricos sobre el comportamiento del sistema.

---

## Tracing

Seguimiento del recorrido de una request dentro del sistema.
